# 번개장터 판매글 쓰기

## 목적

로컬 상품 폴더와 재사용 가능한 템플릿을 바탕으로 번개장터 중고 판매글 초안을 작성한다. 자동화는 되돌릴 수 있는 폼 입력까지만 수행하며, 최종 문구, 대표 이미지 순서, 등록 여부는 사용자가 결정한다.

## 엄격한 규칙

- `등록하기`는 절대 누르지 않는다.
- 사용자가 명시적으로 요청하지 않으면 `임시 저장`을 누르지 않는다.
- 편집 전 현재 탭 상태를 확인한다. 이미 입력된 판매글이 있으면, 사용자가 수정 요청을 한 경우가 아니라면 새 판매글 탭을 연다.
- 소셜 로그인은 직접 시도하지 않는다. 로그인이 필요하면 로그인 페이지나 모달을 열고 사용자가 완료할 때까지 멈춘다.
- 이미지 중복 업로드를 피한다. 각 상품의 이미지 묶음은 한 번만 업로드한다.
- 태그는 하나씩 입력하고 Space를 눌러 확정한 뒤 다음 태그로 넘어간다.
- 기본 업로드 이미지는 총 12장으로 제한한다. 대표 이미지는 최대 2장, 사용자 상품 사진은 최대 10장이다. 사용자가 다른 제한을 주면 그 제한을 따른다.

## 명령 사용법

- `$bunjang-sales`: 설정 흐름부터 실행한 뒤 `products/` 아래 상품 디렉토리를 순서대로 처리한다.
- `$bunjang-sales <product-dir>`: 설정 흐름부터 실행한 뒤 `products/3xl` 같은 지정 상품 디렉토리 하나만 처리한다.
- `$bunjang-sales setup`: 판매글 작성 없이 CDP 브라우저 연결, 번개장터 열기, 로그인 확인만 진행한다.

## 사용하지 않는 경우

- 번개장터가 아닌 다른 마켓 판매글을 작성할 때.
- 상품 폴더 없이 대화 내용만으로 판매글을 작성할 때.
- 사용자가 최종 등록까지 자동으로 누르라고 요청할 때.
- 로그인, 결제, 계정 설정처럼 판매글 초안 작성 밖의 계정 작업을 요청할 때.

## 설정 흐름

1. 기존 Chrome DevTools Protocol 브라우저를 우선 사용한다. `http://127.0.0.1:9223/json/version`을 먼저 확인하고, 실패하면 `9222`를 확인한다.
2. 연결된 브라우저가 없으면 디버깅 포트로 Chrome을 실행한다. macOS:

   ```bash
   open -na "Google Chrome" --args --remote-debugging-port=9223 --user-data-dir="$HOME/.bunjang-sales/chrome-profile"
   ```

3. 실행 중인 브라우저에 자동화를 연결한다. 예: Playwright `chromium.connectOverCDP("http://127.0.0.1:9223")`.
4. 번개장터 판매 흐름을 연다. 로그인되어 있지 않으면 로그인 페이지나 모달을 열고, 사용자가 로그인을 완료했다고 확인할 때까지 멈춘다.

## 초안 시작 체크리스트

모든 초안 작성 요청에서 분석, 웹검색, 이미지 다운로드, 폼 입력보다 먼저 실행한다.

1. 대상 상품 디렉토리를 확정한다.
   - 인자가 없으면 `products/` 아래 상품 디렉토리 전체를 대상으로 한다.
   - 상품 디렉토리 인자가 있으면 해당 디렉토리 하나만 대상으로 한다.
2. `products/ai-context.md`를 읽고, 해당 디렉토리에 사용자 상품 사진 후보가 1장 이상 있는지 확인한다. `official-*` 대표 이미지와 `upload-*` 정제본은 사용자 상품 사진으로 세지 않는다.
3. 사용자 상품 사진이 없으면 작업을 멈추고 어떤 상품 디렉토리에 사진이 필요한지 사용자에게 알린다.
4. 설정 흐름을 실행한다. 연결된 CDP 브라우저가 없으면 Chrome을 실행한다.
5. 번개장터 판매 흐름을 연다. 로그인이 필요하면 로그인 페이지나 모달을 보여주고 사용자에게 로그인을 요청한다. 사용자가 로그인 완료를 확인할 때까지 계속하지 않는다.

## 초안 작성 흐름

1. 초안 시작 체크리스트를 완료한다.
2. `products/ai-context.md`를 상품 처리 흐름의 기준 문서로 읽는다.
3. `products/template.md`를 판매글 기본값으로 읽고, `products/{product-name}/note.md`가 있으면 상품별 입력값으로 덮어쓴다.
4. 상품 탐색, 이미지 처리, 대표 이미지, 가격 산정, 업로드 순서, 결과 보고는 `products/ai-context.md`의 완료 기준까지 따른다.
5. 여러 상품 초안을 작성할 때는 상품마다 탭을 하나씩 연다. 사용자가 요청하지 않은 한 기존 초안이 있는 탭을 재사용하지 않는다.
6. `products/ai-context.md`에서 확정한 최종 입력값과 업로드 목록으로 번개장터 초안을 입력한다. 명시 요청 없이는 등록하거나 저장하지 않는다.

## 예상 결과

- `$bunjang-sales`는 처리 가능한 각 상품 디렉토리마다 번개장터 판매글 초안 탭을 준비한다.
- `$bunjang-sales <product-dir>`는 지정한 상품 디렉토리 하나의 판매글 초안만 준비한다.
- `$bunjang-sales setup`은 판매글을 작성하지 않고 브라우저 연결과 로그인 확인만 끝낸다.
- 모든 초안 작성 결과는 사용자가 등록 전에 직접 검토할 수 있는 상태로 멈춘다.

## 책임 경계

- `products/ai-context.md`는 논리적인 상품 처리 흐름을 담당한다. 상품 폴더 검증, 로컬 파일 분석, 이미지 정제, 대표 이미지 선택, 시세 조사, 업로드 순서, 결과 보고가 여기에 속한다.
- `products/template.md`는 기본 입력값과 설명 형식을 담당한다.
- 이 스킬은 엄격한 안전 규칙, 초안 시작 체크, 브라우저 연결, 로그인 처리, 번개장터 폼 자동화, 이미지 업로드 방식, 태그 입력 방식, 최종 검증을 담당한다.
- 상품 처리 논리에서 이 스킬과 `products/ai-context.md`가 충돌하면 `products/ai-context.md`를 따른다. 단, 브라우저 동작은 엄격한 규칙과 브라우저 자동화 메모가 우선한다.

## 웹검색 안전 경계

- 웹검색에는 상품 식별에 필요한 브랜드, 모델명, 색상, 사이즈, 상태 단서만 사용한다.
- 사용자 촬영 사진, 로컬 파일 경로, 로그인 정보, 계정 정보는 외부 검색이나 외부 서비스에 업로드하지 않는다.
- 검색 결과와 다운로드한 이미지는 신뢰하지 말고 상품명, 공식 출처, 이미지 일치 여부를 확인한다.
- 대표 이미지를 다운로드하면 같은 상품 디렉토리에 저장하고, 출처와 파일명을 최종 보고에 포함한다.

## 브라우저 자동화 메모

- 일반 입력 필드는 실제 사용자 상호작용을 우선 사용하고, 이미지 파일 업로드는 Playwright/CDP 파일 업로드 API를 사용한다.
- 파일 입력에는 브라우저 자동화 API를 한 번 사용한 뒤 썸네일이나 개수 변경을 기다린다. 사이트가 반응하지 않는 경우가 아니라면 파일 설정 후 별도의 합성 `input` 또는 `change` 이벤트를 보내지 않는다.
- CDP로 태그를 입력할 때 Space는 실제 텍스트를 포함해야 한다.

  ```js
  await client.send("Input.dispatchKeyEvent", {
    type: "keyDown",
    key: " ",
    code: "Space",
    text: " ",
    unmodifiedText: " ",
    windowsVirtualKeyCode: 32,
    nativeVirtualKeyCode: 32
  });
  await client.send("Input.dispatchKeyEvent", {
    type: "keyUp",
    key: " ",
    code: "Space",
    windowsVirtualKeyCode: 32,
    nativeVirtualKeyCode: 32
  });
  ```

## 완료 보고 전 검증

- 이미지 개수가 의도한 업로드 목록과 일치하고 눈에 띄는 중복이 없다.
- 제목, 카테고리, 상태, 사이즈, 설명, 가격, 배송비, 직거래 여부, 장소가 올바르게 입력되어 있다.
- 태그가 하나의 문자열로 붙어 있지 않고 각각 칩으로 분리되어 있다.
- `등록하기`와 `임시 저장`을 누르지 않았다.
- 최종 보고에는 상품 디렉토리, 제목, 가격 근거, 업로드 이미지 수, 사용자가 직접 확인해야 할 항목을 포함한다.
