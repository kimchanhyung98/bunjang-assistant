# 시나리오 실행 지침

이 문서는 반복 운영 흐름을 짧게 연결하는 최소 안내서입니다.

## 현재 포함 범위

- 상품 검색과 후보 순위화
- 시세 조회와 판매가 추천
- 상품 상세 조회
- 채팅/관심상품 작업 전 로그인 사전 확인
- 상품 폴더 기반 판매글 초안 작성

## 상품 검색과 후보 순위화

1. 검색어와 가격 조건을 확정합니다.
2. 넓은 목록은 `search.listings`, 후보 선별은 `agent-search-rank`를 사용합니다.
3. listing ID, 가격, 상태 단서, 제외 사유를 요약합니다.

## 시세 조회

1. `references/price.md`를 읽습니다.
2. 동일 상품 키워드로 `search.listings`를 실행합니다.
3. 결과가 부족하면 브랜드, 모델명, SKU, 색상, 사이즈를 나눠 재검색합니다.
4. 이상치를 제외하고 범위/중앙값/추천 판매가를 보고합니다.

## 판매글 초안 작성

1. `references/sales.md`, `references/ai-context.md`, `references/template.md`를 읽습니다.
2. 사용자가 지정한 상품 디렉토리와 사용자 촬영 사진을 먼저 검증합니다.
3. 이미지 정제, 사용자 제공 대표 이미지 확인, 시세 조사, 업로드 순서를 준비합니다.
4. 기본 요청은 제목/설명/태그/가격/업로드 계획을 보고하고 멈춥니다.
5. 사용자가 브라우저 입력을 명시한 경우에만 `references/browser.md`를 따릅니다.
6. 최종 등록과 임시 저장 전 사용자 검토 상태에서 멈춥니다.

## 로그인 필요한 작업

1. 래퍼가 `auth.status`를 먼저 실행하게 둡니다.
2. `login_required`면 로그인 페이지 안내까지만 하고 멈춥니다.
3. 사용자가 로그인 완료를 알려주면 원래 capability를 다시 실행합니다.
