# bunjang 라우팅과 경계

이 문서는 공개 `bunjang` 스킬 내부에서만 참조하는 내부 가이드입니다.

## 공개 표면

- 사용자가 설치하거나 호출하는 스킬은 `bunjang` 하나만 가정합니다.
- 도메인별 지식은 별도 공개 스킬이 아니라 이 스킬 내부 참조 문서로만 유지합니다.
- 설치기와 README는 이 공개 표면을 기준으로 정리합니다.

## 전역 탐색 범위

- 저장소와 의존성 준비 상태 확인
- 인증 상태 확인
- 요청을 번개장터 운영, 시세, 판매글, 브라우저 참조 문서로 라우팅
- capability ID와 수동 전용 경계 확인

## 라우팅 규칙

- “검색해줘”, “상품 찾아줘”, “상세 봐줘”: `marketplace.md`
- “시세 알려줘”, “중고가 얼마야”, “판매가 추천해줘”: `price.md`
- “판매글 작성해줘”, “상품 폴더로 등록 초안 만들어줘”: `sales.md`
- “찜한 상품 가격”, “찜했던거 가격 알려줘”, “관심상품 시세”: `marketplace.md`의 “찜한 상품 가격 보강” 절
- “브라우저 열어”, “로그인”, “폼 입력”: `browser.md`

## 공통 실행 원칙

- 먼저 capability와 로그인 필요 여부를 확인합니다.
- 없는 기능을 사실처럼 말하지 않습니다.
- 수동 전용 작업은 수동 전용으로 보고하고 실행하지 않습니다.
- 판매글 최종 등록과 구매 확정은 항상 사용자 몫입니다.
