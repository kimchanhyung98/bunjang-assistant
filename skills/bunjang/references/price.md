# 시세 조회

이 문서는 “번개장터에서 A 상품 시세 알려줘”, “A 중고가 얼마야”, “판매가 추천해줘” 요청을 처리하는 내부 실행 지침입니다.

## 목표

- 동일/유사 매물의 현재 번개장터 가격대를 확인합니다.
- 이상치를 제외하고 현실적인 시세 범위와 추천 판매가를 제시합니다.
- 근거가 부족하면 부족하다고 말하고, 사용한 검색어와 한계를 남깁니다.

## 기본 흐름

1. 상품명을 구조화합니다.
   - 브랜드
   - 모델명
   - SKU 또는 세부 모델
   - 색상
   - 사이즈
   - 구성품/상태 조건
2. 첫 검색은 사용자의 원문에 가까운 query로 `search.listings`를 실행합니다.
3. 결과가 부족하면 query를 좁히거나 넓힙니다.
   - `브랜드 모델명 SKU`
   - `브랜드 모델명 색상`
   - `모델명 사이즈`
   - 오타/한글/영문 표기 변형
4. 같은 상품이 아닌 결과를 제외합니다.
5. 지나치게 낮거나 높은 가격, 부품/박스만 판매, 상태가 크게 다른 매물을 이상치로 분리합니다.
6. 남은 가격으로 하한/상한, 중앙값 또는 대표 구간을 계산합니다.
7. 판매 목적이면 상태와 구성품을 반영한 추천 등록가를 제시합니다.

## 호출 예시

```bash
npm run bunjang -- search.listings '{"query":"발렌시아가 3XL 42","maxItems":30,"withDetail":true}'
npm run bunjang -- agent-search-rank '{"query":"발렌시아가 3XL 42","maxItems":10}'
```

## 보고 형식

```text
검색어: ...
사용한 매물 수: n개
제외한 결과: n개 (사유: ...)
시세 범위: ...원 ~ ...원
대표 구간: ...원대
판매 추천가: ...원
주의: 사이즈/구성품/상태에 따라 조정 필요
```

## 주의

- 결과가 3개 미만이면 “참고용”이라고 명시합니다.
- 번개장터 검색 결과만으로 정품 여부를 판단하지 않습니다.
- 판매가 추천은 확정 가격이 아니라 등록 시작가입니다.
- `bunjang-cli`가 제공하지 않는 방문자 수, 찜 수, 판매 완료 데이터는 꾸며내지 않습니다.
