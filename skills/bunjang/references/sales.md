# 판매글 초안

이 문서는 “번개장터 판매글 작성해줘”, “번개장터에 A 디렉토리의 상품들 판매글 작성해” 요청을 처리하는 내부 실행 지침입니다.

## 목표

사용자가 지정한 로컬 상품 디렉토리와 스킬 내부 템플릿을 바탕으로 번개장터 판매글 텍스트 초안과 업로드 계획을 준비합니다. 기본 요청인 “판매글 작성해줘”는 브라우저 입력이나 이미지 업로드를 실행하지 않고 초안 보고에서 멈춥니다.

## 기준 파일

- `skills/bunjang/references/ai-context.md`: 상품 처리 규칙
- `skills/bunjang/references/template.md`: 판매글 템플릿
- `사용자가 지정한 상품 디렉토리/note.md`: 선택 메모

## 시작 체크리스트

1. 사용자가 경로를 지정했는지 확인하고, 지정한 경로를 상품 루트 또는 단일 상품 디렉토리로 확정합니다.
2. `skills/bunjang/references/ai-context.md`와 `skills/bunjang/references/template.md`를 먼저 읽습니다.
3. 상품 디렉토리에 사용자 촬영 사진 후보가 1장 이상 있는지 확인합니다.
4. `official-*`, `upload-*`는 사용자 촬영 사진으로 세지 않습니다.
5. 사진이 없으면 브라우저나 웹검색을 시작하지 않고 사용자에게 필요한 파일을 말합니다.

## 디렉토리 해석

- “A 디렉토리의 상품들”처럼 요청하면 A를 상품 루트로 봅니다.
- 상품 루트 바로 아래에 하위 디렉토리가 있으면 각 하위 디렉토리를 개별 상품으로 처리합니다.
- 지정한 디렉토리 자체에 사진과 `note.md`가 있고 하위 상품 디렉토리가 없으면 지정한 디렉토리를 단일 상품 디렉토리로 처리합니다.
- 경로가 불명확하면 저장소의 기본 상품 폴더를 가정하지 말고 사용자에게 실제 경로를 확인합니다.

## 상품 처리 흐름

1. 상품 디렉토리 전체 파일을 확인합니다.
2. 이미지 디코딩 가능 여부와 실제 포맷을 확인합니다.
3. 사용자 촬영 사진, 대표 이미지 후보, 업로드 정제본 후보를 분리합니다.
4. 업로드 가능한 JPEG/PNG가 아니면 `upload-*.jpg` 정제본을 만듭니다.
5. 상품명, 브랜드, 모델명, 사이즈, 색상, 상태, 구성품을 추론합니다.
6. `price.md`를 사용해 시세를 조사합니다.
7. 사용자가 제공한 대표 이미지와 사용자 상품 사진 기준으로 업로드 순서를 만듭니다.
8. 템플릿에 맞춰 제목, 설명, 태그, 가격, 거래 설정을 준비합니다.

## 명시 요청 시 브라우저 입력

사용자가 “번개장터에 입력해줘”, “브라우저로 올려줘”, “사진 업로드까지 해줘”처럼 폼 입력을 명시한 경우에만 브라우저 자동화를 진행합니다. 브라우저 자동화는 [`browser.md`](./browser.md)를 따릅니다.

허용:

- 로그인 페이지 표시
- 판매글 작성 페이지 열기
- 이미지 업로드
- 제목/설명/가격/카테고리/상태/태그/배송 정보 초안 입력

금지:

- `등록하기` 클릭
- `임시 저장` 클릭
- 최종 구매 확정
- 계정 설정 변경
- 소셜 로그인 버튼 직접 클릭

## 완료 보고

- 상품 디렉토리
- 제목
- 가격 근거
- 업로드 이미지 수
- 생성/정제한 이미지 목록
- 불확실한 정보
- 사용자가 직접 확인해야 할 항목
