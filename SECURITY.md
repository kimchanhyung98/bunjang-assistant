# Security Policy / 보안 정책

## Supported Versions / 지원 버전

| Version | Supported          |
|---------|--------------------|
| latest  | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability / 취약점 보고

보안 취약점은 공개 GitHub 이슈나 토론으로 보고하지 마세요. 공개 글에 재현 절차, 토큰, 세션 파일, 계정 정보, 개인 정보를 올리지 마세요.

### How to Report / 보고 방법

1. GitHub Security Advisories를 사용합니다.
    - 저장소의 Security 탭으로 이동합니다.
    - "Report a vulnerability"를 선택합니다.
    - 영향 범위, 재현 절차, 예상 영향, 관련 로그를 포함합니다.

2. 저장소 관리자에게 비공개로 연락해야 하면 GitHub 프로필에 공개된 연락 수단을 사용합니다.
    - 제목에 `SECURITY`를 포함합니다.
    - 민감한 토큰, 쿠키, 세션 파일은 요청 전까지 첨부하지 않습니다.

### Response / 대응 기준

- 접수된 보고는 가능한 한 빨리 확인하려고 합니다.
- 재현 가능한 취약점은 심각도와 수정 계획을 정리해 공유합니다.
- 수정이 완료될 때까지 공개 세부 공개는 보류해 주세요.
